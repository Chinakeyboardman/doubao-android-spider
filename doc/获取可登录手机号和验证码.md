# SMS API 接口文档

## **版本**: v1.0

**基础URL**: `https://sms.guangyinai.com`
**协议**: HTTPS
**认证方式**: Bearer Token

## 认证说明

所有接口请求需在HTTP Header中携带Bearer Token进行身份认证。

### 请求头


|                   |            |        |                                     |
| ----------------- | ---------- | ------ | ----------------------------------- |
| **参数名**           | **类型**     | **必填** | **说明**                              |
| **Authorization** | **string** | **是**  | **Bearer Token，格式: Bearer {token}** |
| **Content-Type**  | **string** | **否**  | **推荐使用 application/json**           |


### 示例

```http
GET /api/phone/get?device_id=device1&platform=yuanbao
Host: sms.guangyinai.com
Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxx
Content-Type: application/json
```

## 接口列表

### 获取手机号

分配一个可用的手机号码用于自动登录。

#### 接口信息


|        |                                   |
| ------ | --------------------------------- |
| **项目** | **说明**                            |
| 接口地址   | /api/phone/get                    |
| 请求方式   | GET                               |
| 请求类型   | application/x-www-form-urlencoded |
| 响应类型   | application/json                  |


#### 请求参数

**Query Parameters:**


|            |        |        |                                          |         |
| ---------- | ------ | ------ | ---------------------------------------- | ------- |
| **参数名**    | **类型** | **必填** | **说明**                                   | **示例值** |
| device_id  | string | 是      | 设备唯一标识符                                  | device1 |
| platform   | string | 是      | 平台名称，可选值: doubao, kimi, qianwen, yuanbao | yuanbao |
| profile_id | string | 否      | 浏览器配置文件ID                                | 1       |


#### 请求示例

```http
GET /api/phone/get?device_id=device1&platform=yuanbao&profile_id=1 HTTP/1.1Host: sms.guangyinai.com
Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxx
```

```bash
curl -X GET "https://sms.guangyinai.com/api/phone/get?device_id=device1&platform=yuanbao&profile_id=1" \
  -H "Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json"
```

#### 响应参数

**成功响应 (200 OK):**


|             |         |                  |                  |
| ----------- | ------- | ---------------- | ---------------- |
| **参数名**     | **类型**  | **说明**           | **示例值**          |
| success     | boolean | 请求是否成功           | TRUE             |
| phoneNumber | string  | 分配的手机号（包含+86国家码） | "+8618813985447" |


**失败响应 (200 OK):**


|         |         |        |                              |
| ------- | ------- | ------ | ---------------------------- |
| **参数名** | **类型**  | **说明** | **示例值**                      |
| success | boolean | 请求是否成功 | FALSE                        |
| error   | string  | 错误信息   | "No phone numbers available" |


#### 响应示例

**成功示例:**

```json
{"success": true,"phoneNumber": "+8618813985447"}
```

**失败示例 - 无可用手机号:**

```json
{"success": false,"error": "No phone numbers available"}
```

**失败示例 - 缺少参数:**

```json
{"success": false,"error": "Device ID is required"}
```

#### 业务逻辑

1. 系统根据 `device_id`、`platform`、`profile_id` 分配一个未被占用的手机号
2. 手机号返回时包含国家码 `+86`
3. **重要**: 使用手机号前需去除 `+86` 前缀
4. 同一个 `device_id` + `platform` + `profile_id` 组合可能返回不同的手机号
5. 若无可用手机号，返回错误信息

#### 注意事项

- ⚠️ 返回的手机号包含 `+86` 前缀，需要去除后再使用
- 手机号格式转换示例: `+8618813985447` → `18813985447`
- 建议在获取手机号后立即检查本地速率限制
- 若手机号被速率限制，应调用"通知手机号占用"接口并重新获取

### 获取短信验证码

获取指定手机号收到的最新短信验证码。

#### 接口信息


|        |                                   |
| ------ | --------------------------------- |
| **项目** | **说明**                            |
| 接口地址   | /api/messages/latest              |
| 请求方式   | GET                               |
| 请求类型   | application/x-www-form-urlencoded |
| 响应类型   | text/plain                        |


#### 请求参数

**Query Parameters:**


|           |        |        |                                |                     |
| --------- | ------ | ------ | ------------------------------ | ------------------- |
| **参数名**   | **类型** | **必填** | **说明**                         | **示例值**             |
| key       | string | 是      | 短信Key，格式: {手机号}_{平台}（手机号不含+86） | 18813985447_yuanbao |
| deviceId  | string | 是      | 设备唯一标识符                        | device1             |
| profileId | string | 否      | 浏览器配置文件ID                      | 1                   |


#### Key格式说明


|                |               |                     |
| -------------- | ------------- | ------------------- |
| **平台**         | **Key格式**     | **示例**              |
| 元宝 (Yuanbao)   | {手机号}_yuanbao | 18813985447_yuanbao |
| 豆包 (Doubao)    | {手机号}_doubao  | 18813985447_doubao  |
| Kimi           | {手机号}_kimi    | 18813985447_kimi    |
| 通义千问 (Qianwen) | {手机号}_qianwen | 18813985447_qianwen |


**注意**: Key中的手机号必须去除 `+86` 前缀。

#### 请求示例

```http
GET /api/messages/latest?key=18813985447_yuanbao&deviceId=device1&profileId=1 HTTP/1.1Host: sms.guangyinai.com
Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxx
```

```bash
curl -X GET "https://sms.guangyinai.com/api/messages/latest?key=18813985447_yuanbao&deviceId=device1&profileId=1" \
  -H "Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxx"
```

#### 响应参数

**成功响应 (200 OK):**

返回6位数字验证码（纯文本格式）


|        |         |         |
| ------ | ------- | ------- |
| **类型** | **说明**  | **示例值** |
| string | 6位数字验证码 | 844714  |


**失败响应:**


|                  |               |
| ---------------- | ------------- |
| **HTTP状态码**      | **说明**        |
| 404 Not Found    | 验证码尚未到达，请稍后重试 |
| 400 Bad Request  | Key格式错误       |
| 401 Unauthorized | 认证失败          |


#### 响应示例

#### 响应示例

**成功示例 (200):**

```plaintext
844714
```

**失败示例 (404):**

```plaintext
Not found
```

#### 业务逻辑

1. 系统根据 `key` 查询最新的短信验证码
2. 验证码格式为6位数字
3. 响应格式为纯文本（非JSON）
4. 404状态码表示验证码尚未到达，需等待后重试

#### 调用时机

1. **发送短信后等待**: 点击"发送验证码"按钮后，**等待45秒**再调用此接口
2. **重试策略**:
  - 如果返回404，等待5秒后重试
  - 建议重试3-5次
  - 每次重试间隔5秒

#### 注意事项

- ⚠️ 必须在点击"发送验证码"按钮后等待 **45秒** 再调用
- ⚠️ Key中的手机号不能包含 `+86` 前缀
- ⚠️ 响应是纯文本，不是JSON格式
- 404响应是正常情况，表示验证码还未到达
- 验证码有时效性，建议获取后立即使用

### 通知手机号占用

通知系统某个手机号被占用（如速率限制），系统将标记该手机号并在后续分配时避免使用。

#### 接口信息


|        |                  |
| ------ | ---------------- |
| **项目** | **说明**           |
| 接口地址   | /api/phone/bid   |
| 请求方式   | POST             |
| 请求类型   | application/json |
| 响应类型   | application/json |


#### 请求参数

**Body Parameters (JSON):**


|             |        |        |                  |             |
| ----------- | ------ | ------ | ---------------- | ----------- |
| **参数名**     | **类型** | **必填** | **说明**           | **示例值**     |
| deviceId    | string | 是      | 设备唯一标识符          | xiaocaopc   |
| phoneNumber | string | 是      | 被占用的手机号（不含+86前缀） | 18813985447 |
| platform    | string | 是      | 平台名称             | yuanbao     |
| profileId   | string | 否      | 浏览器配置文件ID        | 1           |


#### 请求示例

```http
POST /api/phone/bid HTTP/1.1Host: sms.guangyinai.com
Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxx
Content-Type: application/json

{"deviceId": "device1","phoneNumber": "18813985447","platform": "yuanbao","profileId": "1"}
```

```bash
curl -X POST "https://sms.guangyinai.com/api/phone/bid" \
  -H "Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "deviceId": "device1",
    "phoneNumber": "18813985447",
    "platform": "yuanbao",
    "profileId": "1"
  }'
```

#### 响应参数

**成功响应 (200 OK):**


|         |         |        |                       |
| ------- | ------- | ------ | --------------------- |
| **参数名** | **类型**  | **说明** | **示例值**               |
| success | boolean | 请求是否成功 | TRUE                  |
| message | string  | 响应消息   | "Phone marked as bid" |


#### 响应示例

**成功示例:**

```json
{"success": true,"message": "Phone marked as bid"}
```

**失败示例:**

```json
{"success": false,"error": "Invalid phone number"}
```

#### 业务逻辑

1. 系统将该手机号标记为"占用"状态
2. 后续的 `/api/phone/get` 请求将避免分配此号码
3. 占用状态有时效性，通常24小时后自动解除

#### 使用场景

- 手机号触发速率限制时
- 手机号登录失败多次时
- 检测到手机号可能被封禁时

## 常见问题 (FAQ)

**Q1: 为什么API返回的手机号有+86前缀？**

A: API统一返回国际格式（E.164标准），使用时需根据实际场景去除前缀。

**Q2: 获取验证码接口返回404怎么办？**

A: 404表示验证码尚未到达，这是正常情况。建议等待5秒后重试，最多重试3-5次。

**Q3: 如何判断手机号是否可用？**

A: 获取手机号后，应在本地进行速率限制检查（`checkRateLimit`）。如果被限制，调用"通知手机号占用"接口后重新获取。

**Q4: 为什么要等待45秒才能获取验证码？**

A: 短信从平台发送到手机、再由手机上报到API系统需要时间。45秒是经过测试的最优等待时间。

**Q5: 同一个手机号可以在多个平台使用吗？**

A: 可以。不同平台的短信通过Key中的平台后缀区分（如 `_yuanbao`、`_doubao`）。

**Q6: 获取到的手机号会过期吗？**

A: 手机号本身不会过期，但如果被标记为"占用"，24小时后才能再次使用。
