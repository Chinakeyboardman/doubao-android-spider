## 4. 提升点（无引用路径，同类可比）

同为「成分类」无引用题（约千字答案、2–3 屏），本会话三次迭代的分阶段计时：

| 阶段 | 原始版本 | v0.1.0（提速） | 本次（自适应等待） |
|------|:---:|:---:|:---:|
| 展开+数据采集 | **158.2** | ~30 | **23.4** |
| 回答长截图 | 65.0 | ~29 | 26.7 |
| 新会话+切模式+发送 | ~32.7 | ~32 | **29.6** |
| **单题合计** | **≈344** | ≈186 | **168.4** |

**单题总耗时 ≈344s → 168s，降幅约 51%（腰斩）。**

关键改动：

1. **无引用早退**：`_expand_and_collect_panels` 从回答底部一次自适应上滚定位思考/引用面板，顶部确无面板即返回，省去原本的重复上滚 + 两次未压缩 `dump_hierarchy`。该阶段 **158s → 23s**。
2. **面板检测合并**：`_thinking_panel_on_screen` 由 3 次独立 `xpath.get()` 合并为**单次 union XPath**，滚动循环里每轮少抓 2 次 hierarchy —— 长截图阶段 **65s → ~27s**。
3. **自适应等待**（本次）：新建对话 / 切模式 / 回复后 settle 由固定 `sleep` 改为**轮询就绪信号 + 兜底超时**（超时上限 ≤ 原 sleep，最坏不劣于改动前）。「新会话+切模式+发送」由 ~32.7s 降到 29.6s，且**最坏情况不退化**。
